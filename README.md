# SYNQ

SYNQ - учебный fullstack-проект по кейсу «Создание цифровой платформы для подбора участников и формирования команд». Целевая идея продукта по ТЗ - веб-сервис, в котором пользователь заполняет профиль, публикует проекты, ищет людей и собирает команды под конкретные задачи.

Текущий репозиторий соответствует этапу `Спринт 2 / Показ 2`: в центре реализации сейчас находится ядро приложения и аутентификация. Репозиторий собран как две отдельные части (`frontend` и `backend`) и поднимается одной командой через Docker Compose.

На текущем этапе проект запускается локально и демонстрируется через `localhost`. Публичное развертывание упомянуто в ТЗ как требование более позднего, финального этапа, а не как обязательный результат второго спринта.

По требованиям второго спринта пользователь должен:

- зарегистрироваться;
- подтвердить email;
- войти в аккаунт;
- выйти из аккаунта;
- попасть на базовые страницы приложения после авторизации.

На текущем этапе в проекте уже есть:

- регистрация с отправкой кода подтверждения на email;
- подтверждение почты через шестизначный код;
- вход и выход пользователя;
- получение текущего пользователя;
- минимальные страницы после авторизации;
- базовый просмотр и редактирование профиля как задел под следующий спринт;
- PostgreSQL для основных данных и Redis для временного хранения кода подтверждения.

В полном MVP по ТЗ дальше должны появиться:

- полноценные профили с навыками;
- карточки проектов;
- поиск пользователей и проектов;
- механизм заявок на участие в проекте.

## Стек

- frontend: React 18, Vite, React Router;
- backend: FastAPI, SQLAlchemy, Alembic;
- база данных: PostgreSQL 16;
- кеш и временные данные: Redis 7;
- инфраструктура: Docker Compose, Nginx.


## Быстрый старт через Docker

Это основной сценарий, если нужен просто рабочий стенд локально.

1. Создать backend-конфиг:

```bash
cp backend/.env.example backend/.env
```

2. Заполнить как минимум:

- `DB_USER`
- `DB_PASSWORD`
- `DB_NAME`
- `SECRET`
- `ALGORITHM`
- `EMAIL_LOGIN`
- `EMAIL_PASSWORD`
- `SMTP_HOST`
- `SMTP_PORT`
- `code_expires_in`

3. Поднять сервисы:

```bash
docker compose --env-file backend/.env up -d --build
```

После старта приложение будет доступно на [http://localhost](http://localhost).

- frontend: `http://localhost`
- API через nginx: `http://localhost/api/v1`

Остановить и удалить тома:

```bash
docker compose --env-file backend/.env down -v
```
